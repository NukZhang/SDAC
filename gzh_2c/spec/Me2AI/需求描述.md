# 需求描述（Me2AI）

> 只写“需求事实”，不要写实现方案。

## 1) 项目概述
- 项目目标：基于给定原始素材，产出可直接用于公众号发布的高质量二创稿件，并形成标准化生产流程。
- 面向用户：公众号运营者、内容编辑、品牌市场团队。
- 核心价值：在不牺牲事实准确性与品牌调性的前提下，提升内容产能与一致性。

## 2) 核心玩法 / 功能
- 功能列表（按优先级）：
  1. 原文解析与关键信息抽取（事实点、观点、案例、数据）
  2. 公众号二创生成（标题备选、摘要、正文、结尾 CTA）
  3. 风格控制（严谨型 / 亲和型 / 增长型）
  4. 发布辅助（封面文案建议、标签建议、发布时间建议）
  5. 质检清单（事实核对、敏感词检查、结构完整性检查）
- 用户主流程（步骤）：
  1. 提交原文素材与目标受众说明
  2. 选择二创风格与字数区间
  3. 生成初稿并查看事实引用标记
  4. 执行质检并返回问题清单
  5. 输出终稿与发布建议

## 3) 边界与非目标（非常重要）
- 本阶段明确不做：
  - 不接入公众号后台自动发布
  - 不做图片自动生成，仅给出配图建议
  - 不做多账号权限系统
- 不允许 AI 自己加的内容：
  - 不得编造数据、案例、引用来源
  - 不得扩展为短视频脚本平台或多渠道 CMS
  - 不得修改既定的输出格式与验收口径

## 4) 验收口径（必须可验证）
- 通过标准：
  - 输入一篇原文后，可稳定输出 1 份终稿 + 至少 3 个标题备选 + 1 份质检报告
  - 终稿结构完整（标题、导语、主体、结尾 CTA）
  - 事实项可追溯到输入素材，不出现无来源断言
- 必须提供的证据（命令/截图/日志/链接）：
  - 生成命令与终端输出日志
  - 输出文件路径与内容快照
  - 质检报告结果（通过/不通过及原因）

## 5) 迭代划分（可选，但强烈推荐）
- 迭代 1：完成“原文解析 + 初稿生成 + 固定模板输出”。
- 迭代 2：增加“风格控制 + 质检报告 + 发布辅助建议”。
