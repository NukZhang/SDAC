# 技术约束（Me2AI）

> 约束优先级高于实现建议。AI 必须严格遵守。

## 1) 技术栈
- 语言/框架：以 Markdown 文档驱动为核心，脚本语言优先 Python 3.11+。
- 构建工具：如需自动化脚本，使用 `make` 或简单 shell 命令，不引入复杂构建系统。
- 运行环境：Linux 命令行环境，UTF-8 编码，离线可运行为优先。

## 2) 禁止事项（零容忍）
- 禁止引入的依赖：
  - 禁止引入需要付费云服务才能运行的核心依赖
  - 禁止引入重量级 Web 框架（Django、Spring 等）用于当前阶段
- 禁止的架构动作（例如：大重构、换框架、跨模块抽象）：
  - 禁止重写 SDAC 的 `spec/` 治理框架结构
  - 禁止跨目录大规模重命名导致模板不可复用
- 禁止的行为（例如：自创需求、自作主张改约束）：
  - 禁止 AI 根据“经验”擅自新增功能目标
  - 禁止 AI 修改 Me2AI 文档中的业务边界

## 3) 性能 / 兼容性
- 性能指标（如有）：单篇 3000 字以内原文，生成完整终稿时间应在 30 秒内（本地脚本级目标）。
- 兼容性（浏览器/系统/设备）：文档与脚本在 Linux/macOS 终端可用，Windows 通过 WSL 兼容。

## 4) 工程约定
- 目录结构约定：
  - `spec/Me2AI/*` 仅人类维护
  - `spec/AI2AI/AI2AI.md` 仅 AI 维护
  - 输出示例放在 `examples/`（如新增）
- 代码风格（lint/format）：Markdown 标题层级清晰；脚本遵循 PEP8，命令可直接复制执行。
- 测试策略（单测/集成/e2e）：当前以“命令可复现 + 输出可核对”为主，后续可补充自动化测试。
- CI 要求（如有）：当前阶段无强制 CI，但提交前必须完成最小自检（文档结构、路径、命令有效性）。
