# 🧪 Skill 违规检测 Checklist（AICoding 快速纠偏）

> **用途**
> - 当 AI 输出跑偏、越界、过度设计时
> - 用最短路径定位问题并强制回到 Spec 轨道

---

## ✅ 使用方式

在发现问题时，对 AI 追加以下指令即可：

> 请对你刚才的输出执行一次「Skill 违规检测 Checklist」，  
> 标记所有违规项，并在不引入新需求的前提下修正。

---

## 一、Spec 级违规（最高优先级）

- [ ] 是否修改、弱化或扩展了 Me2AI 中的内容  
- [ ] 是否实现了 Me2AI 未定义的功能  
- [ ] 是否跨越了当前迭代编号  
- [ ] 是否在未完成当前迭代前推进后续迭代  

**命中任意一条：必须全部推翻并重做当前迭代**

---

## 二、流程级违规（执行顺序）

- [ ] 是否在分析前未明确当前迭代目标  
- [ ] 是否未对照既有 AI2AI.md  
- [ ] 是否为未来迭代提前设计代码  
- [ ] 是否引入与当前迭代无关的抽象  

---

## 三、Skill 级违规（子 Skill）

### 🗡 Weapon System

- [ ] 武器是否直接修改角色状态  
- [ ] 是否绕过攻击框直接造成伤害  
- [ ] 是否在武器中写 AI 或移动逻辑  
- [ ] 是否引入不必要的武器管理系统  

---

### 🤖 AI System

- [ ] AI 是否直接修改速度或位置  
- [ ] AI 是否直接触发攻击或结算伤害  
- [ ] AI 是否感知或依赖武器内部状态  
- [ ] AI 是否与角色逻辑耦合  

---

### 🔄 State Machine

- [ ] 是否存在隐式状态（if 判断代替状态）  
- [ ] 状态是否承担了 AI 或武器职责  
- [ ] 状态切换是否不可追踪  
- [ ] 是否用状态代替冷却或计时器  

---

## 四、技术约束违规

- [ ] 是否引入外部库或依赖  
- [ ] 是否使用 ES6 module  
- [ ] 是否破坏 Canvas 渲染路径  
- [ ] 是否违背“加速度驱动角色移动”原则  

---

## 五、AI2AI 记录违规

- [ ] 是否未更新 AI2AI.md  
- [ ] AI2AI 中是否出现新需求或设计建议  
- [ ] 是否未客观记录已完成事实  
- [ ] 是否遗漏关键实现限制或已知问题  

---

## 六、结果处理规则（强制）

- 若存在 **Spec 级违规**
  - 放弃当前结果
  - 重新执行当前迭代

- 若存在 **流程 / Skill 级违规**
  - 删除违规实现
  - 回退至最小可完成集

修正后必须：
- [ ] 重新通过本 Checklist
- [ ] 更新 AI2AI.md

---

## 📌 定位说明

- 本 Checklist 不讨论“好不好”
- 只判断“是否违规”
- 用于快速纠偏，而非需求讨论
