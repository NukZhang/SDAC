# Skill 违规检测 Checklist（A/B/C）

> 目的：把“AI 输出是否合规”变成可判定的工程规则。

## A 级（立即中止 + 全回滚）
- 改动/扩展/弱化 Me2AI 内容（含改写措辞）
- 自行新增需求、玩法、模块、规格类型（未获人类允许）
- 跨迭代实现：实现了当前迭代之外的功能
- 引入外部依赖或大规模架构重构（未获 Me2AI 允许）

## B 级（回退到最小可完成集）
- 过度设计：为未来预留扩展点、抽象层、通用框架
- 越界修改：修改了非 Allowed Paths（lane 违规）
- 未提供可验证证据（命令/结果/日志），导致无法验收

## C 级（进入 Minimal Fix Diff）
- 文档/命名不一致、遗漏更新 AI2AI
- 轻微偏离代码风格/工程约定（可通过最小 diff 修复）
- 输出格式不结构化，影响后续自动化消费

## 输出要求
命中时输出：
- 违规等级：A/B/C
- 命中的条目（精确到条）
- 影响范围（文件/模块/迭代）
- 下一步动作（按一键总控流程）
